# Grafana Dashboards

This document covers the configuration and usage of Grafana dashboards for monitoring the Kubernetes cluster using Prometheus metrics.

---

## Prerequisites

* Kubernetes cluster is running and all nodes are in `Ready` state
* Prometheus and Grafana are deployed in the `monitoring` namespace
* Grafana is accessible via Ingress or port-forward
* Prometheus data source is automatically configured by `kube-prometheus-stack`

Verify monitoring components:

```
kubectl get pods -n monitoring
```

## Access Grafana

Access Grafana using the configured Ingress:

```
http://<INGRESS-IP>
```

OR (optional port-forward for debugging):

```
kubectl port-forward -n monitoring svc/monitoring-grafana 3000:80
```

```
http://localhost:3000
```

## Retrieve Grafana Admin Password

```
kubectl get secret -n monitoring monitoring-grafana \
-o jsonpath="{.data.admin-password}" | base64 --decode && echo
```

## Import Kubernetes Dashboards

Dashboards are imported from Grafana Labs using dashboard IDs.

### Kubernetes Cluster Monitoring Dashboard

Dashboard ID:

```
315
```

Steps:

1. Navigate to **Grafana → + → Import**
2. Enter the dashboard ID
3. Select **Prometheus** as the data source
4. Click **Import**

---

### Kubernetes Cluster Overview Dashboard

Dashboard ID:

```
6417
```

Repeat the same import steps using the ID above.

## Node Exporter

Node Exporter is deployed automatically as part of the `kube-prometheus-stack` and runs as a DaemonSet on all nodes.

Verify Node Exporter pods:

```
kubectl get pods -n monitoring -l app.kubernetes.io/name=node-exporter
```

---

## Alertmanager

Alertmanager is deployed automatically with the Prometheus stack and is responsible for handling alerts generated by Prometheus.

Verify Alertmanager pods:

```
kubectl get pods -n monitoring -l app.kubernetes.io/name=alertmanager
```
